# Moltbook Phase Transition Decision Note

## Purpose
This note defines when the passive Moltbook research mission should remain read-only, and when it would make sense to upgrade to a public participation phase.

## Current Default
Current status: **Phase 1 only**

That means:
- passive observation
- no account
- no posting
- no commenting
- no public identity footprint

## Why This Boundary Exists
Public participation is not just research. It creates:
- a visible external identity
- reputational risk
- expectation of follow-up replies
- time cost from low-signal engagement
- ambiguity about representation and voice

Because of that, public presence should only happen if it clearly improves learning quality enough to justify the added risk and maintenance burden.

## Criteria for Switching to Phase 2
Move from Phase 1 to Phase 2 only if **at least 3** of the following are true:

### 1. High Signal Density
Moltbook consistently produces technically meaningful discussions with concrete details about:
- tools
- frameworks
- architecture
- deployment
- failures
- multi-agent coordination

### 2. Unique Signal
The platform yields information that is not easily available elsewhere from:
- GitHub
- X
- blog posts
- docs
- Discord communities

### 3. Repeatable Patterns
There are recurring patterns worth tracking over time, such as:
- repeated framework choices
- repeated coordination strategies
- repeated failure modes
- repeated workflow bottlenecks

### 4. Real Operators Present
There is evidence that people running real agents or production-adjacent systems are active there, not just posting hype or mockups.

### 5. Public Questions Would Improve Learning
There are clear situations where a short technical question or comment would likely surface better information than passive observation alone.

### 6. Reasonable Time-to-Insight Ratio
The time spent reading Moltbook produces better insight per hour than alternative sources.

## Hard Stop Conditions
Do **not** move to Phase 2 if any of these are true:
- content is mostly hype or vague demos
- discussions rarely contain implementation detail
- follow-up burden would exceed value
- the platform appears inactive or low quality
- public presence would create confusion about representation
- the likely ROI is lower than other available research channels

## Recommended Phase 2 Shape
If Phase 2 is approved, keep it minimal:
- create one account
- use a restrained profile
- publish one short intro post
- comment only on high-signal threads
- avoid volume posting
- avoid social engagement loops

## Review Questions
Before switching to Phase 2, answer these explicitly:

1. What exact signal are we missing by staying passive?
2. Why is Moltbook the best place to obtain that signal?
3. What is the expected ROI of public participation?
4. What is the maximum acceptable time budget per week?
5. What is the rollback plan if the platform turns out low-value?
